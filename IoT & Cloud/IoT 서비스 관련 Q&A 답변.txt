영상 관련 기능 정리_이채림
[영상 스트리밍]
라즈베리 파이에서 찍은 프레임을 mqtt통신으로 web에 전달해 보여줌. (안드로이드 webview로 확인)

+혹시 물어본다면....
-스트리밍 속도 높이기: 전송속도를 높이기 위해 랜선을 사용
-카메라 2개 쓴걸들킨다면..?
     : 하나는 스트리밍용 하나는 영상 저장용으로, 스트리밍용은 해상도를 낮춰 실시간으로 스토커의 행동을 끊김없이 볼 수 있게 구현하였다.
     : 네트워크 속도가 빨라진다면 카메라 하나로도 충분히 구현가능.
-영상 비율: 저장용(900x1024: AI반에서 세로가 더 길게 찍히기를 희망), 스트리밍용(320*240)

[영상 저장]
-움직임 감지 센서에 움직임이 감지되면
	:영상 저장(1분)이 시작(영상은 10프레임)
	: AI모델에게 시작 신호 전달
	: 현재 날짜와 현재시각을 읽어 S3버킷에 폴더 생성
-프레임을 mqtt통신으로 전달해 S3버킷에 jpg파일로 저장한다.
-S3버킷에 프레임이 전달되는 동안 AI모델에서 이상행동으로 판단시
	:matt로 신호 전달해 안드로이드 상단 알림(이상행동 발생)
	:모델 중지
-영상 저장(1분) 완료시
	:저장된 프레임을 mp4파일로 변환
	:해당 영상이 이상행동 영상인지의 유무에 따라 삭제 또는 보관

+영상 저장 관련된 추가 사항
*우리는 모델이 4개이기 때문에 한 영상에 대해 최대 4개의 이상행동 판단이 내려질 수 있음. 그렇지만 알림이 4번 오는것은 상당히 불편한 일이므로 대책을 마련
*모델에서 이상행동으로 판단 시 해당 영상프레임이 담긴 폴더 내에 stranger.txt파일이 생성됨
*AI모델이 이상행동으로 판단하면, 폴더 내 stranger.txt파일의 여부를 확인해 파일이 없으면 stranger.txt파일을 새로 생성 후 안드로이드에 알림 신호 전달
*폴더 내 stranger.txt파일이 존재한다면 무시한다.(이미 앱 알림도 갔고 영상이 이상행동영상으로 판단된 것이기 때문이다.)

[영상확인]
-web에서 S3버킷 내 존재하는 mp4파일명을 읽어옴
-읽어온 파일명을 클릭시 해당 mp4파일이 실행됨
-해당 web은 안드로이드 webview에서 확인 가능
